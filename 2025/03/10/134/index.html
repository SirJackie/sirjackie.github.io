3. 错误一：CORS 跨域名访问

按F12打开控制台，我们发现，Cross origin requests被拦截导致界面加载失败：

这是由于新版Chromium / Firefox内核的CORS安全措施引起的，会拦截所有非本域名的访问，实现域名之间Data隔离，保障用户安全。

但对于本地文件，没有域名，于是全部被当成CORS跨域名访问，所以全部被禁止了。

4. 错误二：错误的MIME标识

具体来说，服务器返回每一个静态文件时，都会返回一个MIME标识，表明文件的具体类型。

而这里，浏览器想要一个JS文件，服务器却返回一个 text/plain 文本文档标识，不符合，故浏览器拒绝加载。

经过排查，发现是由于Python命令 python -m http.server 80 启动的是简单服务器，不能正确处理MIME头，需要更换一个更完善健壮的服务器，方案如下：

  1. 部署一个Nginx / IIS / Node.js 静态HTTP服务器
    • 但这就和我们的初衷相违背了。我们希望简单打包，方便客户启动，结果装HTTP服务器更加麻烦了！
  2. 使用Python写一个简单服务器，再用Python打包成Exe
    • 这个很好！客户双击直接启动，无需安装Python(因为打包了)，也不需要配置复杂的HTTP服务器
