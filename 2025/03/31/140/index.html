Stable Diffusion WebUI 完全配置教程

缘起

之前一直使用绘世大佬的SD整合包，一路顺风，感觉没有自己研究配置的必要。

但随着SD绘画探索的深入，手上积累了2-3个整合包，里面各有不同的模型，不同的存档文件；同时为了分别在两边都能使用同一个模型，要复制两次，久而久之就产生了200-300GB的SD整合包文件，非常占用内存。

完全配置的目的，就是从零实现SD整合包的配置，从而达到：

  • 离线迁移：可以在无网环境下，在一台全新的电脑上安装Stable Diffusion
  • 目录管理：熟悉目录结构，可以合并整合包，导入/导出存档，按需复制需要使用的模型，从而优化结构，减少硬盘占用

话不多说，我们直接开始！

4.2. 备份 / 还原 我们的生成的作品，图片 + 参数

经过我的研究，发现，Stable Diffusion已生成作品的保存方式如下：

  • 提示词数据，并不是存在数据库 / JSON / TOML中，而是直接存在PNG输出图片内部，作为文件附加信息
  • 因此，我们不需要担心提示词数据，只需要备份 / 还原PNG图片文件即可。
  • 文生图 / 图生图 / 重绘 / 超分，全部存在 outputs 文件夹中
  • 图片查看器——收藏夹中的作品，全部存在 log\images 文件夹中
  • 只需要拷贝 outputs 和 log 文件夹，即可实现备份 / 还原 我们的生成的作品！

4.5. 删除敏感缓存数据

该方法一般不需要用到，仅作为补充说明，属于是探索过程中的意外收获。

有时，你需要删除你的Stable Diffusion中的所有资料，恢复到干净的原样。

以下是需要删除的资料：

我会详细解释，每个资料的作用：

  • log/images：一个文件夹，存放SD收藏夹中的图片，需要删除！
  • outputs/*：多个文件夹，分别存放SD文生图、图生图的结果图片，需要删除！
  • extensions/stable-diffusion-webui-images-browser-***/iib.db：图片管理器的次要数据库文件，包含敏感信息，需要删除！
  • extensions/stable-diffusion-webui-images-browser-***/wib.sqlite3：图片管理器的数据库文件，包含敏感信息，需要删除！
  • python/Lib/site-packages/gradio/launches.json：一个计数器文件，记录启动的次数，每次启动后，里面的值都会增加1，可删可不删。

4.6. 备份敏感缓存数据

该方法一般不需要用到，仅作为补充说明，属于是探索过程中的意外收获。

着重备份如下资料：

  • log/images：一个文件夹，存放SD收藏夹中的图片
  • outputs/*：多个文件夹，分别存放SD文生图、图生图的结果图片
  • extensions/stable-diffusion-webui-images-browser-***/iib.db：图片管理器的次要数据库文件
  • extensions/stable-diffusion-webui-images-browser-***/wib.sqlite3：图片管理器的数据库文件

需要还原时，只需把所有这些文件粘贴覆盖进去，即可100%原汁原味还原之前的工作数据，很方便！

5. 总结：目录结构

  • extensions：插件
  • outputs 和 log：作品文件
  • models：模型文件
