使用Visual Studio Comunity 2019开发Unity C#脚本没有自动补全的解决方法

最近开始试着玩Unity3D，要为场景中的物体编辑脚本。Unity3D推荐的脚本语言是C#，在Unity打开C#就会使用Visual Studio来进行编辑。

启动Visual Studio之后注意到，Unity类和方法名都没有代码补全，而且Unity似乎也编译不过：

在网上找了一圈，发现并没有人遇到这样的问题，那可能性比较大的应该就是环境问题了。

我之前先安装过Visual Studio进行C/C++开发，之后才安装Unity。初步断定是我的Visual Studio没有C#语言和Unity类的支持。

解决方法

1.打开Visual Studio Installer

2.找到你要安装Unity支持的VS版本，点击“更多”——“修改”

3.在弹出的子窗口中选勾“使用Unity的游戏开发”，之后点击“修改”

4.Visual Studio Installer会开始连接服务器下载支持组建，等待它下载完成：

5.完成之后，一定要重启Unity，Unity会开始编译一些脚本

6.启动完成，在点击C#脚本进入Visual Studio，会发现代码补全出现了：

原来无法编译通过是因为CharacterController少打了一个l，哭笑不得。

7.更正代码，保存之后回到Unity，会发现已经没有编译错误了：
