Git 仓库分支名字 如何从main改回master

有时候，我们推送Git仓库到远端，使用SourceTree的一键推送功能，结果创建出来的Main Branch名称叫做 main，而不是我们更习惯、更约定俗成的 master，如何解决？

解决方案如下：

  • 代码层面解决：在本地仓库创建master并推送到远端
  • GitHub / Gitee 网页端解决：删除远端的main分支，并且将远端主分支切换到新建的master分支

2. 网页端解决

登录到代码托管平台（如 GitHub、GitLab），进入仓库的设置页面，将默认分支改为 master。

2.1. GitHub

进入GitHub页面，我们会发现，现在main和master两个分支并存：

因此，我们需要删掉其中一个分支。

我们尝试用代码的方式，删除远端的main分支，不出意料的被拦截了，因为触发了GitHub的保护机制：

因此，我们需要在网页端操作。点击View All Branches：

尝试点击垃圾桶，删除main，结果被拦截，因为main还是远端的默认分支，无法被删除：

经过一番研究，发现了方法：

  • GitHub管理很严，不允许删除主分支，并且不允许重新指定主分支
  • 但是GitHub允许重命名主分支
  • 因此，我们的策略是：删除除了main主分支外的所有分支，然后将main重命名为master，即可解决问题

点击main右侧三点水——Rename branch：

将名字改为master：

点击确定，可以看到，我们的任务搞定了：

那么你可能会说：我们刚才敲代码，岂不是没有意义了？此言差矣，Gitee会用到。

2.2. Gitee

Gitee方法很不同，经过一番研究，发现了方法：

  • Gitee，不允许删除主分支，并且似乎没办法重命名主分支
  • 但是Gitee可以允许重新指定主分支
  • 因此，我们的策略是：重新制定主分支为master，然后删除main分支

进入Gitee的分支管理界面，点击“切换默认分支”：

然后将默认分支切换为master：

最后，删除main即可：

3. 收尾工作

我们完成了重命名，还要记得收尾。因为本地仓库还是有master和main两个分支，所以要解决：

  • 最好的方法，是重新从网站中，git clone一个完成的，只包含一个分支的仓库
  • 也可以尝试，手动把main分支删除，但是我没有测试过可行性
  • 推荐使用第一种方法

至此，我们完成了分支的重命名工作！
