SDL2 完全配置教程

引言

SDL2并不好配置，以下这些步骤缺一不可：

5个必须步骤：

  • 引入Include Dir

  • 引入Lib Dir（x86 x64）

  • 引入附加Lib文件（SDL.lib）

  • 将DLLs拷贝到编译后Exe旁边

  • int main 必须有 argc argv 否则无法编译

2个可选步骤：

  • 工程配置：/MT /MTd
  • 工程配置：Release x64 释放性能

本文将以详细的描述和截图，讲清楚如何在VS2019中配置SDL2。（VS2022也类似，大同小异）

0. 获取SDL并解压

请确保您在GitHub Release中下载到如下压缩包：

  • 一共5个部件，分别是：SDL2 SDL2_ttf SDL2_image SDL2_mixer SDL2_net
  • 这5个部件分别位于5个独立的GitHub工程中，分别实现了：基本、帧缓冲、图像加载、音频播放、网络连接
  • 你可以选择自己需要的部分，引入项目，不一定要全部引入
  • 为什么有10个压缩包？因为我下载了Linux + Windows两个版本的，VC给Windows，mingw给Linux

将5个 VC 版本的压缩包解压，得到5个文件夹：

进入每个文件夹，都有类似的文件结构：

  • README-xxx：代表说明文档（用不到）
  • docs：代表参考文档（用不到）
  • cmake：使用CMake进行编译所需的文件（用不到）
  • 等等

其中，只有2个文件夹需要用到：

  • include：存放头文件
  • lib：存放链接文件/DLL文件

如图所示：

接下来，我们需要做一个很重要的步骤，“分类提取”：

  • 自己建立3个文件夹：Include、Lib_x86、Lib_x64、DLLs_x86、DLLs_x64
  • 将所有 .h 文件，扔到 Include
  • 将所有 x86 (32位) 的 .lib/.pdb 文件，扔到 Lib_x86
  • 将所有 x64 (64位) 的 .lib/.pdb 文件，扔到 Lib_x64
  • 将所有 x86 (32位) 的 .dll 文件，扔到 DLLs_x86
  • 将所有 x64 (64位) 的 .dll 文件，扔到 DLLs_x64

1. 引入Include Dir

  • VS2019——项目——右键——属性
  • 选择“所有配置”——“所有平台”
  • C/C++——常规——附加包含目录——编辑

  • 将刚才我们创建的 Include 目录加进去，然后点击确定——应用

2. 引入Lib Dir（x86 x64）

  • 选择“所有配置”——“Win32”
  • 链接器——常规——附加库目录——编辑

  • 将 Lib_x86 目录添加进去，点击确定——应用

  • 选择“所有配置”——“x64”
  • 链接器——常规——附加库目录——编辑

  • 将 Lib_x64 目录添加进去，点击确定——应用

3. 引入附加Lib文件（SDL.lib）

  • 选择“所有配置”——“所有平台”
  • 链接器——输入——附加依赖项——编辑

  • 将 LibList.txt 中的所有文字，复制粘贴到文本框，然后点击确定——应用

  • 成功后如图所示：

5. int main 必须有 argc argv 否则无法编译

SDL有一个奇怪的特性，那就是 main 主函数，必须写成带有参数的形式：

如果写成最简洁的 main 主函数形式，会造成编译失败！

所以，如果出现编译失败，不一定是Include或者Lib的问题，可以检查一下 main 函数是否正常！

6. 工程配置：/MT /MTd

  • 如果不进行配置，对于有安装VS2019的系统，可以正常打开编译成功的Exe
  • 但如果系统没有安装过VS2019，例如用户的系统，就会无法启动程序，报错 找不到vc_xxx.dll！
  • 问题的根源，是程序依赖于 vc_xxx.dll，而没有安装VS2019的电脑上，会缺少这个DLL
  • 解决方法：我们在编译时，手动设置将 vc_xxx.dll 作为静态链接库，直接嵌入到我们的程序中，解决麻烦
  • 具体方法如下：
  • 选择“Debug”——“所有平台”
  • C/C++——代码生成——运行库——下拉菜单
  • 把**“多线程调试 DLL (/MDd)”，改为“多线程调试 (/MTd)”**，取消对DLL的依赖

  • 完成后如下：

  • 选择“Release”——“所有平台”
  • C/C++——代码生成——运行库——下拉菜单
  • 把**“多线程 DLL (/MD)”，改为“多线程 (/MT)”**，取消对DLL的依赖

  • 完成后如下：

原理介绍：

  • 运行库分为4种，/MT /MTd /MD /MDd
  • /MT 和 /MD 的区别：Multi-Threading 和 Multi-Threading DLL 的区别，是否依赖DLL
  • 有d 和 没d 的区别：是否支持Debug模式。如果有d，是给Debug用的，如果没d，是给Release用的

7. 工程配置：Release x64 释放性能

  • 编译时，尽量选择“Release x64”，以获得最佳性能：

  • 原因：“Release”是发布模式，移除优化了很多调试用的汇编指令，比“Debug”模式运行速度快，性能高
  • 原因：“x64”是64位模式，能够利用大于4GB部分的内存，而“x86”是32位模式，只能使用<=4GB的内存
