Git Bash / SourceTree 选哪个好

作为一个有多年使用经验的Git用户，分享一下自己的体验：

Git Bash

优点：

  • 方便的命令行操作：如果你使用命令行编辑器（如vim）写代码，使用Git Bash可以帮助你把所有工作流都放在命令行，无需切换界面。
  • 利于自动化：因为Git Bash使用CLI，所以可以利用这个特性编写自动化脚本。例如：一次批量Push五十个仓库到GitHub；统计50个仓库的总Commit数，等等。

缺点：

  • 难以处理分支：在命令行里面处理3分支、4分支错综复杂的网状结构，简直痛苦至极。

SourceTree

优点：

  • 比命令行更方便的图形化界面：实测发现这比输入一堆git status, git commit -m xxx, git push命令快很多，只要点几个按钮就可以
  • 很简洁明了的分支图，易于处理多分支分叉、合并等复杂任务。

缺点：

  • 学习成本：需要花1-2天的时间熟悉界面，如果习惯用命令行，一开始可能会不适应。
  • 难以自动化：编写脚本点击按钮，比编写脚本执行命令麻烦。

我的选择

目前，我：

  • 使用SourceTree作为主力，方便处理多分支问题（以前觉得没必要图形化，用了之后发现真的香）
  • 使用Git Bash处理复杂的问题，或者编写自动化脚本（少用Git Bash，多用SourceTree）
  • 自己一个人编写的项目，尽量少用分支，不然给自己找罪受。
