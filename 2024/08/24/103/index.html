树莓派快速入门：软件篇

最近打算为公益图书馆捐赠一批树莓派，需要提前配置好，方便小朋友们使用，遂写下这篇博客，记录配置方法。

本篇博客是两篇姊妹篇中的第二篇：软件篇；您可以在我的往期博客中，查阅第一篇硬件篇。

1. 初始化系统

在官网下载 img 格式的系统镜像文件后，用 Win32DiskImager 即可刷入SD卡。

下载、刷入过程很简单，这里就不过多赘述，有不明白的，您可以查阅网上的资料。

初始化过程之后，才是真正会遇到挑战的阶段。首先就是选择语言了。如果您选择了中文，就需要进行2hr的漫长等待，因为系统会联网下载中文语言包，而国际网速很慢：

您也可以使用魔法进行加速，但很麻烦，需要对路由器施加魔法。我不介意，就等待2hr即可。

一旦初始化完毕，进入系统界面，您就可以用软件施加魔法，这样就简单了。所以需要熬过漫长的第一次等待。

2. 解决4个HDMI驱动引起的问题

我使用一个7寸HDMI小型显示器，作为树莓派的屏幕。可惜进入系统后不久，我就遇到了4个奇怪的问题：

  • 一旦播放视频，会马上黑屏，过一会儿重新亮起，但分辨率崩坏不正常
  • 一旦播放视频，会马上黑屏，然后HDMI屏幕自带的扬声器，会失效无法使用
  • 关机很慢，一直卡在 Power Off 一行字很久
  • 开始菜单点击“注销”后，重新登录，会发现桌面崩溃黑屏，只剩下任务栏

Bug的视频展示：


关机很慢，一直卡在 Power Off 一行字很久（试过能卡40min，什么鬼）：

一开始，遇到这4个问题，我百思不得其解，尝试过各种方法：

  • 卸载重装HDMI驱动，更换HDMI线缆，发现线缆并没有质量问题，都一样
  • 尝试修改 /boot/firmware/config.txt 配置文件，手动设定锁死分辨率，无效
  • 禁用了GPU驱动和OpenGL Driver，改而使用MESA软渲染驱动，结果直接黑屏
  • 黑屏导致系统无法启动，只好完全重装系统来修理，又花了一个半小时

但是，问题都无法得到解决！花了3-4hr，都找不到问题，有一种想要把树莓派退货的冲动！

最后，发现了问题的解决方法，意外的简单，简单得让人吃惊：

  • 所有4个问题，都是因为**使用了带有扬声器的HDMI屏幕**导致的！
  • 解决问题的方法：插入3.5mm有线耳机，到耳机孔
  • 然后，所有问题就都消失了，一切迎刃而解！
  • 黑屏、声音、无法关机、桌面崩溃，看起来毫不相关的4个问题，实际上都是由于HDMI驱动对HDMI自带扬声器支持不好导致的，改用耳机即可解决！

于是，果断额外下单一批耳机，加入到捐赠清单，搞定！即解决了系统崩溃问题，还能解决小朋友们使用电脑，声音外放，互相干扰的问题，一举两得！

3. 日常使用技巧

  • 发现了切换中英文的快捷键：Ctrl + 空格，和 Windows 自定义的 Ctrl + Shift 不一样。
  • 发现了截屏快捷键：按 PrtSc 键，结果不会进入剪贴板，而是会保存在用户主目录里面的“图片”文件夹，这也和 Windows 的剪贴板模式不一样。

4. 解决录屏失败问题

需要给小朋友们准备公益编程课的课件，操作录屏必不可少。

本来想使用好用的 OBS Studio 进行录屏，但发现OBS并没有在官方软件源中，下载麻烦，而且据说OBS在树莓派上很消耗性能，不能流畅运行，遂放弃OBS。

一番寻找，决定使用 SimpleScreenRecorder 这款软件，它好评如潮，简单易用。

开始使用 SimpleScreenRecorder 进行录屏，结果失败，只能录到声音，录不到画面，画面黑屏。

后来发现，这时由于窗口管理器不支持录屏所引起的。窗口管理器出来背锅：

情况如下：

  • 树莓派默认使用Wayland进行窗口管理，Wayland比较小众，不支持录屏。
  • 我们把Wayland换成更通用的X11，即可解决问题！

搞定了，启用了X11，结果发现了一些意料之外的额外好处和红利：

  • 帧率变高，更加流畅！
  • 窗口大小调整更完善，即使我用1024*600的小屏幕，也不会出现窗口过大，超出屏幕，并且无法点击关闭按钮的情况了！（使用Wayland，关闭按钮出现在屏幕边界之外，无法用鼠标点击，非常烦人！）
  • Chrominum 终于不会出现无法弹出的 Bug！（使用Wayland时，点击 Chrominum 右上角——三点水——书签，是无法正常弹出书签窗体的，无法浏览书签和历史记录。更换成X11之后，这个问题就得到了解决！）
