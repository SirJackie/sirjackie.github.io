Git / SourceTree 克隆仓库的所有分支

现在，我有一个远程仓库，具有两个分支：

当我试图用SourceTree将它克隆时，出现了问题：

问题：克隆完之后只有master分支

如上图所示，本地branches里面只有master。

有几种可能性：

  • git只下载了master分支
  • git下载了所有分支，但只是显示了master分支

事实证明，后者是对的。我们可以做一个实验：取消勾选Show Remote Branches，看看分支图中的workBranch是否仍然存在：

这就证明了：git下载了所有分支，但只是显示了master分支。

解决方法：通过Checkout远端分支，来显示隐藏分支

Checkout之后，我们发现，新的分支出现了，问题解决。

你可能会发现：新的节点并没有出现，只是原来的“Feature 3c”节点多了一个分支名。

所以，Checkout与否的区别只在于：

  • Checkout：能切换到隐藏分支的Commit，能显示隐藏的分支名，能够在原来的隐藏分支上提交新的Commit。
  • 不Checkout：能切换到隐藏分支的Commit，但是不能显示隐藏的分支名，也不能够在原来的隐藏分支上提交新的Commit。

所以，建议在仓库克隆之后，Checkout所有远端分支。

新问题：远端分支列表中，多出一个不是我们创建的HEAD分支

解决方法：HEAD是最后活动分支的别名，不用刻意删除它

我们尝试对HEAD分支进行Checkout，发现git提示：HEAD分支不是一个合法分支，这是为什么呢？

我们使用git branch --all命令来查看所有分支：

可以看到，HEAD分支只是master分支的别名，并不是一个真实存在的分支。我们只需要无视它就可以了，不需要刻意去删除它，也实际上无法删除。
